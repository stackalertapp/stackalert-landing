FAQ

Frequently asked questions.

Is it safe to give StackAlert access to my AWS account?
Yes. StackAlert uses a read-only IAM role — it can only read Cost Explorer data, never make changes. The exact Lambda function is open-source on GitHub. We follow least privilege: the role is scoped to only the Cost Explorer API calls we need.
Does it tell me which AWS service caused the spike?
Yes — that's the whole point. StackAlert monitors each AWS service independently (EC2, Bedrock, S3, RDS, Lambda, etc.). When Bedrock spikes 400% but EC2 is stable, you see exactly that — not just "total bill went up 12%." This is what AWS Budgets and Cost Anomaly Detection don't give you at a glance.
I already have AWS Budgets set up. Why do I need this?
AWS Budgets sends emails when you cross a monthly threshold — that's too late. StackAlert checks every 6 hours, detects unusual patterns early, and delivers context (which service, which region, what changed) directly in Telegram — instant push, not buried in email.
Is my billing data stored in the EU? (GDPR)
Yes. All data is processed and stored in AWS Frankfurt (eu-central-1). We are GDPR compliant and do not transfer data outside the EU. A Data Processing Agreement (DPA / AVV) is available on request.
