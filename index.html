FAQ

Frequently asked questions.

Is it safe to give StackAlert access to my AWS account?
Yes. StackAlert uses a read-only IAM role — it can only read Cost Explorer data, never make changes. The exact Lambda function is open-source on GitHub. We follow least privilege: the role is scoped to only the Cost Explorer API calls we need.
What AWS services does StackAlert monitor?
All AWS services by default — EC2, RDS, Lambda, S3, CloudFront, Data Transfer, and everything else. You can configure alerts per service, per region, or per resource tag. The daily digest always shows your top 5 cost drivers.
I already have AWS Budgets set up. Why do I need this?
AWS Budgets sends emails when you cross a monthly threshold — that's too late. StackAlert checks every 6 hours, detects unusual patterns early, and delivers context (which service, which region, what changed) directly in Slack where your team already works.
Is my billing data stored in the EU? (GDPR)
Yes. All data is processed and stored in AWS Frankfurt (eu-central-1). We are GDPR compliant and do not transfer data outside the EU. A Data Processing Agreement (DPA / AVV) is available on request.
We use AI coding agents (Claude, Cursor, Devin). Will StackAlert catch their costs?
Yes — and this is one of the most critical use cases in 2026. AI coding agents autonomously create AWS resources, trigger Bedrock API calls, and can loop indefinitely if something goes wrong. StackAlert monitors all AWS services including Amazon Bedrock, Lambda invocations, EC2, and Data Transfer. If an agent spins up unexpected resources or enters a loop, you'll know within 6 hours — not at the end of the month. The 7-day rolling average baseline means even a first-time agent spike gets caught immediately.
